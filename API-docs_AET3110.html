Modernized Reference

Overview & Setup

The 3110.net API enables remote control of 3110/3110A units over a LAN using TCP sockets. The distribution includes a RemoteControlClient.dll class library and an example utility with source code.

Integration flow

Your App → RemoteControlClient.dll → 3110/3110A (TCP, port 8182)

Project requirements

  • Add a reference to RemoteControlClient.dll (include .deps.json and .pdb).
  • Default TCP port: 8182.
  • Use a static IP or DHCP reservation for the device.
  • Recommended ResponseTime ≥ 150 ms for reliable polling.
  • If your code uses System.IO.Directory, alias the API type: 
    using Directory = _3110A_Remote_Control.RemoteControlClient.Directory;

Core Properties & Methods

CurrentState (Property)

Returns the device’s read-only State structure.

FieldTypeDescription
StoppedboolTrue if player is stopped.
SystemStatusstringIDLE, PAUSED, or RUN.
SequencestringCurrently playing sequence number.
PlayingSegmentstringCurrent segment number.
SystemOffsetstringValue from Settings → System Calibration.
SystemGainstringValue from Settings → System Calibration.
AmpStatusstringStatus of attached amplifier (returns Run if none).
SerialNumberstringSerial number of the connected unit.

RequestCurrentState() (Method)

public State RequestCurrentState()

Queries the 3110 and populates CurrentState. Use for status UIs; avoid over-polling.

RequestDirList(pathName) (Method)

public List<Directory> RequestDirList(string pathName)

Returns the contents of the given directory; start from RootNode.

Disconnect() (Method)

Stops client communication with the device. Call before exiting your app.

File Operations

  • List Files: Build the device’s file structure once at initialization (≈50 seconds).
  • Load File: Select a test file and load it; the front panel confirms the load.
  • Run Loaded File: After loading, Play, Pause, and Stop behave as on the front panel.

Cookbook: Read the 3110’s Current State

Goal → Prereqs → Steps → Gotchas → Validation → Full example

Use this when

  • You need to poll for status (RUN/PAUSED/IDLE, sequence, segment, gain/offset, amp status, serial number).
  • You want to drive UI updates or gate actions (e.g., enable Play only when allowed).

Prereqs

  • 3110/3110A on the same LAN; default TCP port 8182.
  • Static IP or reservation recommended.
  • Project references RemoteControlClient.dll (include .deps.json/.pdb).
  • Alias Directory type if you use System.IO.Directory.

Steps

  1. Connect to the device

    var client = new _3110A_Remote_Control.RemoteControlClient("10.123.8.44", 8182);
client.ResponseTime = 200; // ≥150 ms recommended

  2. Get the current state

    var state = client.RequestCurrentState();
Console.WriteLine($"Status: {state.SystemStatus} | Seq: {state.Sequence} | Seg: {state.PlayingSegment}");
Console.WriteLine($"Gain: {state.SystemGain} | Offset: {state.SystemOffset} | SN: {state.SerialNumber}");

  3. Drive UI logic

    var isStopped = state.Stopped;
var canPlay   = isStopped || state.SystemStatus == "PAUSED";

  4. Poll on an interval

    var timer = new System.Timers.Timer(500);
timer.Elapsed += (_, __) => {
  try {
    var s = client.RequestCurrentState();
    UpdateUiFromState(s); // your UI update method
  } catch (Exception ex) {
    ShowTransientWarning($"State poll failed: {ex.Message}");
  }
};
timer.AutoReset = true;
timer.Start();

  5. Clean up

    timer?.Stop();
client.Disconnect();

Gotchas

  • Do not change the device’s LAN EOL setting (breaks the client).
  • Use a single timer; don’t allow overlapping polls.
  • Prefer static IP or reservation; DHCP churn causes confusion.
  • Build the file tree once at startup; it can take ~50s.

Validation checklist

  • SerialNumber matches front panel.
  • SystemStatus mirrors the front panel state.
  • During run, Sequence and PlayingSegment change over time.

Full example (Console)

using System;
using System.Timers;
using Directory = _3110A_Remote_Control.RemoteControlClient.Directory;

class Program
{
  static _3110A_Remote_Control.RemoteControlClient client;
  static Timer poll;

  static void Main()
  {
    client = new _3110A_Remote_Control.RemoteControlClient("10.123.8.44", 8182)
    { ResponseTime = 200 };

    poll = new Timer(500);
    poll.Elapsed += (_, __) =>
    {
      try
      {
        var s = client.RequestCurrentState();
        Console.WriteLine($"{DateTime.Now:HH:mm:ss} | {s.SystemStatus} | Seq {s.Sequence} | Seg {s.PlayingSegment} | Gain {s.SystemGain} | Off {s.SystemOffset}");
      }
      catch (Exception ex)
      {
        Console.Error.WriteLine($"Poll error: {ex.Message}");
      }
    };
    poll.AutoReset = true;
    poll.Start();

    Console.WriteLine("Polling. Press ENTER to quit.");
    Console.ReadLine();

    poll.Stop();
    client.Disconnect();
  }
}

Cookbook: Load a Test and Run It (with Pause Default Voltage)

Goal → Prereqs → Steps → Gotchas → Validation

Use this when

  • You want to select a stored test on the 3110/3110A, run it, and control Play / Pause / Stop from software.
  • You need Pause to hold a user‑selected default voltage and resume cleanly.

Prereqs

  • Device powered and reachable on the LAN.
  • Static IP or reservation recommended.
  • Project references RemoteControlClient.dll.

Concept

On first connect, build the device file tree (~50s). Then select and load a test file; Play/Pause/Stop function as on the front panel. When Paused, output drops to the default voltage; Play while Paused resumes at the start of the current segment.

Steps

  1. Connect and initialize

    var client = new _3110A_Remote_Control.RemoteControlClient("10.123.8.44", 8182);
client.ResponseTime = 200; // tune per network
    Tip: Build the file structure once at startup; avoid rebuilding on every action.

  2. (Optional) List files programmatically

    var root = client.RootNode;                       // read-only root
var entries = client.RequestDirList(root.path);   // List<Directory>
// Traverse children to find your test file, then load via your own routine

  3. Load a test (example utility workflow)

    • Open Files → List Files to view the TreeView.
    • Navigate to the desired test file and click Load File.
    • Confirm on the device front panel; controls are now enabled.

  4. Run and control playback

    • Play: starts the test (from idle or paused).
    • Pause: output drops to your default voltage; Play while Paused resumes at the start of the current segment.
    • Stop: halts the test; select and load another file as needed.

  5. (Optional) Hold DC at end of test

    Add a final segment with the required DC level and set its trigger to True so it persists until triggered off.

Validation checklist

  • Front panel shows the selected file after Load File.
  • Play runs; Pause holds your default voltage; Play resumes at segment start.

Gotchas

  • Controls do nothing? Ensure a file is loaded first.
  • Slow startup UI? That’s the initial file tree build (~50s). Do it once.
  • Connection flakes/timeouts? Use ResponseTime ≥ 150 ms; avoid DHCP churn.
  • Do not alter the device LAN EOL; it breaks the client.
